## ūüĎč What is this?

This repository documents my structured journey to master full-stack web development ‚ÄĒ from the very first HTML tag all the way to deploying containerized Node.js applications.

This is not a course. This is not a bootcamp. This is a **personal, self-paced, deeply intentional study plan** where every single topic is learned from scratch, understood technically, and immediately applied through a small focused project.

Every topic I study gets its own folder here with:
- My notes
- The mini-project I built
- Anything interesting I discovered along the way

---

## ūüß† The Study Philosophy

### Small steps. Every day. No skipping.

Most people learn web development by jumping between YouTube videos, copying code they don't understand, and moving on before things really click. I wanted the opposite of that.

My approach has three rules:

**1. Start from zero on every topic**
Even if I've seen something before, I revisit it like it's new. True understanding comes from first principles, not familiarity.

**2. Learn it three ways**
Every topic is studied as: *a simple human explanation ‚Üí a technical deep dive ‚Üí a working mini-project*. You don't really know something until you can explain it simply AND build something with it.

**3. One topic at a time, fully**
No moving forward until the current topic is solid. No multitasking between concepts. Depth over speed, always.

## ūüß© How Each Session Works

Every topic I study follows the same structure:

```
1. Plain English explanation    ‚ÄĒ understand it like a human
2. From scratch setup           ‚ÄĒ first file, first command, first output
3. Technical deep dive          ‚ÄĒ proper terms, internals, gotchas
4. Why this exists              ‚ÄĒ problem it solves, when to use it
5. Alternatives & comparisons   ‚ÄĒ interview-ready trade-off knowledge
6. Runnable code example        ‚ÄĒ complete, commented, copy-paste ready
7. Interview questions          ‚ÄĒ 5 real questions with answers
8. Mini-project                 ‚ÄĒ build something with it today
```

No topic is marked done until I've built the mini-project.
